Práctica 3

  1. Escriba una expresión en Caml cuyo valor sea siempre el mismo que:


-    abs x  (sin utilizar la función abs)
-    not x   (sin usar el operador not)
-    string_of_bool b  (sin utilizar la función string_of_bool)
-    Char.lowercase c   (sin usar la función Char.lowercase)
-    x mod y  (sin utilizar el operador mod)
-    x && (y || z)  (sin usar los operadores && y ||)

  1. Escriba en ocaml definiciones "alternativas" para las funciones abs, lowercase y  string_of_bool, como si no estuviesen predefinidas.


  2. Defina funciones  hd: 'a list -> 'a, tl: 'a list -> 'a list, length:‘a list -> int  y  reverse:‘a list ->‘a list, que se comporten como las homónimas del módulo List, sin usar ninguna de las funciones definidas en este módulo.

     

  3. Defina las funciónes  lmax:‘a list ->‘a  y   lmin:‘a list ->‘a  que devuelvan respectivamente el mayor y el menor valor de los que aparecen en una lista.

  4. Defina una función  rango:‘a list ->‘a *‘a   tal que si  rango l = (a,b)  entonces  lmin l = a   y   lmax l = b.

  5. Defina una función primeros: int -> int list, de forma que primeros n sea la lista ascendente de los n primeros enteros positivos; es decir, la lista de los enteros mayores que 0 y menores o iguales que n. Así, por ejemplo, primeros 4 sería la lista [1; 2; 3; 4].

  6. Defina una función from_to : int -> int -> int list, de forma que from_to m n sea la lista ascendente de los enteros mayores o iguales que m y menores o iguales que n. Así, por ejemplo, from_to 3 7 sería la lista [3; 4; 5; 6; 7].

  7. Defina una función ordenada: 'a list -> bool que indique si una lista tiene sus elementos ordenados según el orden (<=). Así, por ejemplo, ordenada [2; 5; 5; 7]  valdría true, pero ordenada ['a'; 'u'; 'i']  valdría false.
